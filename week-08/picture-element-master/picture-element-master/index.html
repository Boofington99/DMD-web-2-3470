Responsive Images & Backgrounds

Modern web design requires serving optimized images across a vast range of devices and screen sizes. This page demonstrates three techniques for responsive images: the <picture> element, responsive background images using background-size, and the CSS image-set() function for high-DPI displays.

1 The <picture> Element

Assignment Task

Implement a responsive <picture> element using the images provided in the images folder.

  • Display art-large.jpg when the viewport is at least 945px wide
  • Display art-extra-large.jpg when the viewport is at least 1100px wide
  • Provide a fallback image (art-medium.jpg) for smaller screens
  • Use the proper order of <source> elements

2 Background-Size (Contain vs Cover)

Create two containers demonstrating the difference between background-size: contain and background-size: cover.

  • Use images from the images folder (bg-small, bg-medium, bg-large)
  • Switch images using CSS media queries
  • Default image: bg-small.jpg
  • 700px breakpoint: bg-medium.jpg
  • 1000px breakpoint: bg-large.jpg
  • Both containers must use background-position: center and background-repeat: no-repeat

Contain: entire image visible

Cover: fills container

3 Image-Set for High-DPI Displays

Use the CSS image-set() function to load different images based on screen pixel density.

  • pattern-1x.jpg for standard displays (1x)
  • pattern-2x.jpg for retina displays (2x)
  • pattern-3x.jpg for ultra-high-density displays (3x)
